## 二、项目概述

### 2.1 项目背景

运营在 **Temu平台** 日常上新时，需要填写"货号"字段（即内部商家编码）。Temu平台下发备货单时，通过货号与仓库ERP系统的商品数据进行关联，仓库人员据此知晓备货单应发哪些商品。

由于每天运营大量上新，ERP管理员需要为多个团队**每天手动创建大量货号商品**。当前操作模式为：
- 运营在飞书多维表格中维护商品信息（品牌、产品名称、容量、配件等）
- ERP管理员人工在快麦ERP系统中逐一创建贴纸商品和套装商品
- 重复录入工作量大，效率低，且容易出错

### 2.2 项目目标

**核心目标**：构建一个批量建货号工具，支持从飞书多维表格导入数据，自动调用快麦ERP接口完成批量创建，并回填货号到原表格。

**具体目标**：
| 目标编号 | 目标描述 | 成功指标 |
|----------|----------|----------|
| T1 | 实现Excel数据解析与批量导入 | 支持标准模板一键导入，解析成功率≥99% |
| T2 | 自动调用快麦ERP接口创建商品 | 调用成功率≥95%，支持失败重试 |
| T3 | 自动回填货号到表格 | 回填准确率100% |
| T4 | 简化运营操作流程 | 将单批次操作时间从30分钟降低至3分钟以内 |

### 2.3 项目范围

**本期范围内**：
- 支持从飞书多维表格导出的Excel模板导入
- 支持自动创建"贴纸"和"套装"两种商品类型
- 支持调用快麦ERP HTTP接口创建商品
- 支持货号回填到飞书多维表格
- 支持批量操作和单条操作
- 支持操作日志记录和失败重试

**本期范围外**（后续迭代）：
- 直接与飞书多维表格API打通（免导出导入）
- 支持其他ERP系统对接
- 支持图片自动上传
- 支持多平台（非Temu）商品创建

## 三、需求分析

### 3.1 用户画像

| 用户角色 | 角色描述 | 核心诉求 | 使用频率 |
|----------|----------|----------|----------|
| ERP管理员 | 负责在快麦ERP系统中创建和维护商品数据，管理多个团队的货号 | 快速批量创建货号，减少重复操作，降低出错率 | 每天多次 |
| 运营人员 | 在Temu平台上新商品，维护商品基础信息 | 希望货号能快速创建并回填到表格，便于上新使用 | 每天多次 |
| 仓库人员 | 根据备货单发货 | 需要准确的货号与ERP商品关联 | 间接使用 |

### 3.2 用户痛点

1. **重复录入量大**：每天需要为多个团队创建数十甚至上百个货号，每个货号包含贴纸+套装两个商品，人工录入耗时耗力
2. **编码规则复杂**：货号编码需遵循 `69-品牌编码-产品名称简写+自增序号` 规则，人工生成容易出错
3. **套装配置繁琐**：套装商品需要关联多个单品SKU，人工选择容易遗漏或选错
4. **数据回填困难**：创建完成后需要手动将货号回填到飞书表格，容易遗漏
5. **缺乏状态跟踪**：不知道哪些货号已创建、哪些失败、失败原因是什么

### 3.3 使用场景

**场景一：日常批量建货号（高频）**
> 运营每天从飞书多维表格导出待创建商品清单（Excel），交给ERP管理员。管理员使用本工具导入Excel，一键批量创建贴纸和套装商品，系统自动回填货号到表格，记录每个商品的处理状态。

**场景二：单条补建货号（中频）**
> 某个商品遗漏或创建失败，管理员使用工具单条录入信息，快速补建货号。

**场景三：失败重试（低频）**
> 批量创建过程中部分商品因网络或接口异常失败，管理员查看失败列表，一键重试失败的商品。

## 四、业务流程

### 4.1 整体业务流程

```
┌─────────────────┐     ┌──────────────────┐     ┌─────────────────┐
│  飞书多维表格    │────▶│  批量建货号工具   │────▶│  快麦ERP系统    │
│  (商品数据源)    │     │  (本工具)         │     │  (创建商品)     │
└─────────────────┘     └──────────────────┘     └─────────────────┘
         │                       │                       │
         │                       │                       │
         │                       ▼                       │
         │              ┌──────────────────┐             │
         │              │  处理状态记录     │             │
         │              │  + 失败重试       │             │
         │              └──────────────────┘             │
         │                       │                       │
         ▼                       ▼                       ▼
┌─────────────────┐     ┌──────────────────┐     ┌─────────────────┐
│  表格回填货号    │◀────│  结果汇总输出     │◀────│  快麦返回货号   │
│  (商品货号列)    │     │  (Excel/表格)    │     │  (商家编码)     │
└─────────────────┘     └──────────────────┘     └─────────────────┘
```

### 4.2 详细操作流程

**Step 1 — 数据准备**
1. 运营在飞书多维表格维护商品信息（品牌、产品名称、容量、配件、图片URL等）
2. 导出为Excel模板（固定格式）
3. 将Excel文件导入到本工具

**Step 2 — 数据解析与校验**
1. 工具读取Excel文件，解析每行商品数据
2. 自动关联"配件表"和"品牌配置表"，获取单品编码
3. 校验必填字段完整性（品牌、产品名称、容量、配件等）
4. 预览待创建商品列表，标注异常数据

**Step 3 — 生成商家编码**
1. 按规则自动生成贴纸商家编码和套装商家编码
2. 编码规则：`69-品牌编码-产品名称简写+自增序号`
3. 检测编码是否重复（与ERP系统已有编码比对）

**Step 4 — 批量创建商品**
1. 先创建"贴纸"商品（分类固定为贴纸）
   - 商家编码：贴纸编码
   - 名称：品牌 + 产品名称 + 容量 + "贴纸"
   - 图片：有则上传，无则忽略
2. 再创建"套装"商品（类型：不含sku套件）
   - 主商家编码：自定义不重复编码
   - 名称：品牌 + 产品名称 * 1 + 自粘袋 * 1 + …
   - 成分/执行标准：表格提供
   - 套件单品：表格配件 + 上面新增的贴纸单品

**Step 5 — 结果回填**
1. 快麦ERP返回创建结果（商家编码即货号）
2. 将货号回填到表格"商品货号"列
3. 记录每行数据的处理状态：成功/失败/待处理
4. 失败数据记录失败原因，支持一键重试

## 五、功能需求

### 5.1 功能模块总览

| 模块编号 | 功能模块 | 优先级 | 说明 |
|----------|----------|--------|------|
| FM-01 | Excel导入与解析 | P0 | 核心功能，支持模板导入和字段解析 |
| FM-02 | 数据校验与预览 | P0 | 导入后校验数据完整性，预览待创建列表 |
| FM-03 | 编码自动生成 | P0 | 按规则生成贴纸编码和套装编码 |
| FM-04 | 批量创建商品 | P0 | 调用快麦ERP接口创建贴纸和套装 |
| FM-05 | 结果回填与导出 | P0 | 回填货号到表格，导出结果文件 |
| FM-06 | 失败重试 | P1 | 支持失败记录一键重试 |
| FM-07 | 配置管理 | P1 | 管理配件表、品牌配置表等基础数据 |
| FM-08 | 操作日志 | P1 | 记录操作日志，便于追溯和审计 |
| FM-09 | 用户认证 | P0 | 快麦ERP登录态管理和接口鉴权 |

### 5.2 详细功能需求

#### FM-01 Excel导入与解析

**FM-01-01 模板导入**
- 支持拖拽上传Excel文件（.xlsx/.xls格式）
- 支持最大文件大小：10MB
- 支持最大行数：1000行（超出提示分批导入）
- 解析过程显示进度条

**FM-01-02 字段映射**
- 自动识别标准模板字段（品牌、产品名称、容量、配件等）
- 支持非标准模板的手动字段映射
- 支持字段映射配置保存（下次自动应用）

**FM-01-03 数据关联**
- 自动关联"配件表"，将配件名称映射为单品编码
- 自动关联"品牌配置表"，将品牌名称映射为品牌编码
- 关联失败的数据标注为异常，供人工确认

**FM-01-04 导入记录**
- 保存每次导入的文件名、时间、行数、成功/失败数
- 支持查看历史导入记录

#### FM-02 数据校验与预览

**FM-02-01 必填校验**
- 校验字段：品牌、产品名称、容量、配件（至少一项）
- 缺失必填字段的行标红，提示具体缺失字段

**FM-02-02 数据格式校验**
- 容量字段：支持数字+单位（如100ml、50g）
- 品牌字段：需在品牌配置表中存在
- 配件字段：需在配件表中存在（多个配件用逗号/换行分隔）

**FM-02-03 预览列表**
- 以表格形式展示待创建商品列表
- 显示：序号、品牌、产品名称、容量、预计贴纸编码、预计套装编码、状态
- 支持排序、筛选、搜索
- 支持勾选/取消勾选部分商品（仅创建勾选项）

**FM-02-04 异常处理**
- 异常数据支持在线编辑修正
- 支持删除异常行（不参与创建）
- 支持导出异常数据（修正后重新导入）

#### FM-03 编码自动生成

**FM-03-01 编码规则配置**
- 默认规则：`69-品牌编码-产品名称简写+自增序号`
- 支持自定义前缀（如固定"69"）
- 支持品牌编码映射（如wkau→39）
- 支持产品名称简写规则（取首字母或自定义映射）

**FM-03-02 自增序号管理**
- 按品牌维度维护自增序号计数器
- 支持查看每个品牌当前最大序号
- 支持手动调整序号（应对跳号或回退场景）
- 支持预设置编码（跳过自增，使用指定编码）

**FM-03-03 编码预览**
- 在创建前展示每个商品的预计贴纸编码和套装编码
- 支持编码冲突检测（与已有编码重复时提示）

#### FM-04 批量创建商品

**FM-04-01 创建流程**
1. 先批量创建所有"贴纸"商品
2. 再批量创建所有"套装"商品（套装引用已创建的贴纸）
3. 每步操作支持暂停/继续

**FM-04-02 接口调用**
- 调用快麦ERP【新增普通商品】接口
- 支持从Cookie获取Token或手动输入Token
- 支持通过快麦开放平台OAuth授权获取Token
- 接口调用间隔控制：避免触发频率限制（默认500ms/次）

**FM-04-03 创建参数映射**

| 快麦接口字段 | 数据来源 | 说明 |
|-------------|----------|------|
| 商品名称 | 品牌+产品名称+容量+"贴纸"（贴纸） / 品牌+产品名称*1+配件*1+…（套装） | 自动拼接 |
| 商家编码 | 自动生成的编码 | 贴纸编码 / 套装编码 |
| 分类 | 固定"贴纸"（贴纸） / 固定"不含sku套件"（套装） | 配置项 |
| 图片 | 表格提供的图片URL | 有则上传，无则忽略 |
| 套件单品 | 配件编码+贴纸编码 | 从配件表关联 |
| 成分 | 表格提供 | 套装必填 |
| 执行标准 | 表格提供 | 套装必填 |

**FM-04-04 并发控制**
- 支持设置并发数（默认5个，最大20个）
- 接口失败时自动降级为单线程重试

**FM-04-05 创建进度**
- 实时显示总进度：已完成/总数
- 显示当前创建的商品名称和编码
- 支持取消未完成的创建任务

#### FM-05 结果回填与导出

**FM-05-01 结果记录**
- 每条商品记录处理状态：待处理/创建中/成功/失败
- 成功的记录保存返回的商家编码（货号）
- 失败的记录保存错误码和错误信息

**FM-05-02 回填方式**
- 方式一：生成结果Excel文件，用户手动复制货号到飞书表格
- 方式二（后续迭代）：直接调用飞书多维表格API回填

**FM-05-03 结果导出**
- 支持导出结果Excel文件，包含：原始数据+货号+状态+失败原因
- 支持仅导出失败数据（便于修正后重试）
- 支持导出创建成功清单（便于核对）

#### FM-06 失败重试

**FM-06-01 失败分类**
- 网络超时：连接超时、读取超时
- 接口错误：参数错误、重复编码、权限不足
- 系统错误：ERP系统异常、服务不可用

**FM-06-02 重试策略**
- 网络超时：自动重试3次（间隔1s/2s/4s指数退避）
- 接口错误：不自动重试，需人工修正后重试
- 系统错误：提示用户稍后重试

**FM-06-03 重试操作**
- 支持一键重试所有失败项
- 支持勾选部分失败项重试
- 重试前支持修正参数（如修改编码）

#### FM-07 配置管理

**FM-07-01 品牌配置表**
- 管理品牌名称与品牌编码的映射关系
- 支持增删改查
- 支持导入导出
- 示例：wkau → 39

**FM-07-02 配件配置表**
- 管理配件名称与单品编码的映射关系
- 支持按品牌分类管理
- 支持批量导入
- 示例：自粘袋 → SKU-001，说明书 → SKU-002

**FM-07-03 编码规则配置**
- 配置编码前缀（默认69）
- 配置产品名称简写规则
- 配置分隔符（默认"-"）

**FM-07-04 分类配置**
- 配置贴纸固定分类
- 配置套装固定分类
- 配置套装类型（不含sku套件）

#### FM-08 操作日志

**FM-08-01 日志内容**
- 记录每次操作的时间、用户、操作类型、影响行数
- 记录每条商品的创建过程（请求参数、响应结果）
- 记录异常和错误信息

**FM-08-02 日志查询**
- 支持按时间范围、操作类型、商品名称筛选
- 支持导出日志

#### FM-09 用户认证

**FM-09-01 Token管理**
- 支持从快麦ERP Cookie中提取Token
- 支持手动输入Token
- 支持Token有效期检测和自动刷新（如接口支持）
- Token本地加密存储

**FM-09-02 权限校验**
- 创建前校验Token有效性
- Token失效时提示重新获取

## 七、数据需求

### 7.1 导入数据模板结构

| 字段名 | 字段类型 | 必填 | 示例 | 说明 |
|--------|----------|------|------|------|
| 品牌 | 字符串 | 是 | wkau | 需存在于品牌配置表 |
| 产品名称 | 字符串 | 是 | 补色膏 | 用于生成名称和编码简写 |
| 容量 | 字符串 | 是 | 100ml | 数字+单位 |
| 配件 | 字符串 | 是 | 自粘袋,说明书 | 多个配件用逗号分隔，需存在于配件表 |
| 成分 | 字符串 | 否 | 水、甘油... | 套装必填 |
| 执行标准 | 字符串 | 否 | QB/T 1857 | 套装必填 |
| 图片URL | 字符串 | 否 | https://... | 贴纸可选，有则上传 |
| 自定义编码 | 字符串 | 否 | 69-39-BSG07 | 如指定则跳过自动生成 |

### 7.3 编码生成规则

```
贴纸编码: 69-{品牌编码}-{产品简写}{自增序号}
套装编码: 69-{品牌编码}-{产品简写}{自增序号}-T

示例:
品牌: wkau → 编码: 39
产品名称: 补色膏 → 简写: BSG
自增序号: 07 (该品牌下第7个)

贴纸编码: 69-39-BSG07
套装编码: 69-39-BSG07-T
```

### 7.4 状态枚举

## 八、技术方案

### 8.1 系统架构

```
┌────────────────────────────────────────────┐
│              用户界面层 (UI)                   │
│  - Web页面 (Vue/React) 或 桌面客户端 (Electron)│
└────────────────────────────────────────────┘
                    │
┌────────────────────────────────────────────┐
│              业务逻辑层                       │
│  - Excel解析模块                               │
│  - 数据校验模块                               │
│  - 编码生成模块                               │
│  - 接口调用模块                               │
│  - 结果处理模块                               │
└────────────────────────────────────────────┘
                    │
┌────────────────────────────────────────────┐
│              数据层                          │
│  - 配置表存储 (JSON/本地数据库)                │
│  - 导入记录存储                                │
│  - 操作日志存储                                │
└────────────────────────────────────────────┘
                    │
┌────────────────────────────────────────────┐
│              外部接口层                        │
│  - 快麦ERP HTTP API                         │
│  - 飞书多维表格API (后续迭代)                  │
└────────────────────────────────────────────┘
```

### 8.2 技术选型建议

| 层级 | 方案A (Web) | 方案B (桌面客户端) |
|------|-------------|-------------------|
| 前端 | Vue3 + Element Plus | Electron + Vue3 |
| 后端 | Node.js / Python FastAPI | 本地Node.js运行时 |
| 数据存储 | IndexedDB / localStorage | SQLite / JSON文件 |
| Excel解析 | SheetJS (xlsx.js) | SheetJS (xlsx.js) |
| HTTP请求 | Axios | Axios |
| 打包 | 静态页面部署 | Electron Builder |

**推荐方案**：方案B（桌面客户端）
- 理由：工具主要面向内部ERP管理员使用，无需公网部署；桌面客户端可以更好地操作本地文件（Excel导入导出）、安全存储Token、提供更好的原生体验。

### 8.3 接口对接方案

**快麦ERP接口**
- 接口地址：`https://open.kuaimai.com/...`（新增普通商品接口）
- 认证方式：Cookie Token 或 开放平台OAuth Token
- 请求方式：POST
- 数据格式：JSON
- 关键参数：商品名称、商家编码、分类、套件单品等

**接口调用流程**
```
1. 用户登录快麦ERP或提供Token
2. 工具校验Token有效性
3. 按批次调用创建接口（贴纸先创建，套装后创建）
4. 处理接口响应，记录成功/失败
5. 失败项按策略重试
```

### 8.4 数据安全

- Token本地加密存储（AES-256）
- 不存储用户账号密码
- 操作日志脱敏处理
- 支持手动清除所有本地数据

## 九、非功能需求

### 9.1 性能需求

| 指标 | 目标值 |
|------|--------|
| Excel导入解析 | 1000行 ≤ 3秒 |
| 单条商品创建 | ≤ 1秒（含接口调用） |
| 50条商品批量创建 | ≤ 60秒 |
| 界面响应 | 操作反馈 ≤ 100ms |
| 内存占用 | 正常使用 ≤ 500MB |

### 9.2 兼容性需求

- 支持Windows 10/11、macOS 10.15+、Linux主流发行版
- 支持Chrome、Edge、Firefox最新2个版本（如采用Web方案）
- 支持Excel 2010+ 导出的 .xlsx 格式

### 9.3 可靠性需求

- 异常崩溃后支持恢复未完成的任务
- 自动保存操作进度，刷新页面不丢失
- 接口超时默认重试3次
- 支持断网检测，提示用户恢复网络后重试

### 9.4 易用性需求

- 首次使用提供引导教程
- 关键操作（创建、重试）需二次确认
- 提供操作提示和错误说明（中文）
- 支持快捷键（Ctrl+O导入、Ctrl+S导出等）

## 十、风险与依赖

### 10.1 风险分析

| 风险编号 | 风险描述 | 影响程度 | 应对措施 |
|----------|----------|----------|----------|
| R1 | 快麦ERP接口变更或限频 | 高 | 预留接口适配层，关注接口文档更新，限频时自动降速 |
| R2 | 编码重复导致创建失败 | 中 | 创建前预检测编码冲突，支持自定义编码跳过自增 |
| R3 | 飞书表格模板字段变更 | 中 | 支持灵活字段映射，模板版本管理 |
| R4 | 用户Token失效 | 中 | Token有效期检测，失效前提醒，支持快速重新获取 |
| R5 | 配件表/品牌表数据不全 | 中 | 导入时检测并提示缺失配置，支持在线补录 |
| R6 | 网络不稳定导致批量失败 | 低 | 自动重试+断点续传，支持失败后重试 |
